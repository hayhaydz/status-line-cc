# Contributing

Thanks for your interest in improving this project.

## Development Setup

```bash
bun install
bun run build
```

## Tests

```bash
bun test
```

## Pull Requests

- Keep changes focused and well-explained.
- Update or add tests when behavior changes.
- Avoid committing build artifacts (`build/`, `dist/`).
- Do not commit secrets or local config files (`.env`, `.claude/settings.local.json`).

## Reporting Issues

Include:
- What you expected vs. what happened
- Steps to reproduce
- OS and Bun version
- Relevant logs or fixture input
